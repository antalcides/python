Gnuplot

This small tutorial is far from a complete guide to Gnuplot. My intention is to show you how you can use Gnuplot with Spice and LaTeX.

What is Gnuplot?

Gnuplot is a command-driven interactive function and data plotting program. At first sight it might seem a bit unhandy but after a while everyone will have to admit it's a powerfull tool that can make life very easy!
The "gnu" in Gnuplot has nothing to do with GNU (GNU's not Unix). It's a coincidence these letters appear at the beginning of the program's name.

Making data ready for Gnuplot

When you perform a simulation with Spice, you get an outputfile. This file contains a list of plot data but also alot of other (useful) information.
Gnuplot only needs the plot data and fails to work with Spice output files that contain other other things, so we need to clean up that output file first. For this purpose I've written a small program, called spice2data, that you can use in a pipe. To use this program you have to compile it first on your system. Minimal information about this is in the sourcecode itself.

Let's assume you compiled this spice2data and put it in a directory that is in your PATH. You also made a Spice input file that works fine and contains a .PRINT statement: simul.input. To get the outputfile called simul.data for Gnuplot you need to do: spice < simul.input | spice2data > simul.data.

Playing around at the command line of Gnuplot

Now, start Gnuplot and type this: plot 'simul.data'.

Instantly, there will be appearing a window that shows you the plot of your data. Indeed: this is simple! In fact all the work that follows is only to fancy up the output.
I'll give you the commands I used to make my plot. If some things are not clear you can do "help". Gnuplot's help is very good.

 
set grid xtics mxtics ytics mytics linewidth 1
set xlabel '$V_D(V)$'
set label  '$I(mA)$' at 1,.5
set mxtics 10
set format y "$%.0s$";
set mytics 5
set ticscale 0.5
plot 'simul.data' t "  spice"

 

Making the output suitable for inclusion in LaTeX documents

Now we have been playing around at the command line and set things so the plot looks like we want it to look like, it's time to put the commands in a file and add three more lines: 

 
set terminal pslatex
set output "simul.tex"
set grid xtics mxtics ytics mytics linewidth 1
set xlabel '$V_D(V)$'
set label  '$I(mA)$' at -40,.016
set mxtics 10
set format y "$%.0s$";
set mytics 5
set size 2,2.2
set ticscale 0.5
plot 'simul.data' t "  spice"

The first two lines tell Gnuplot to plot in LaTeX format and store the plot in simul.tex.
The set size command needs a bit more explanation. Like you can read in help size, this command stretches the plot both horizontally and vertically. I found that (2,2.2) looks fine if you want a landscape plot that fits on one A4 page and (1.3,3) looks fine if you want a portrait plot that's on one A4 page.

  • Update april '03: changed terminal type from latex to pslatex !
    As a consequence, you have a broader choice of line and point types while it's still possible to use the LaTeX math mode in legends and labels.
    TIP: use the gnuplot commando "test". It shows a testplot with all line and point types for the selected terminal type.
